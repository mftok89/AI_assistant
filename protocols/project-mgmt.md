# プロトコル: プロジェクト管理

タスクの正は Notion タスクDB「AI秘書タスク」。プロジェクトの実体（成果物・資料・メモ）はこのリポジトリの `projects/` で管理する。

## 新規登録

新規の依頼を受けたら、作業開始前にプロジェクトを立ち上げる:

1. 既存プロジェクトの続きでないか `projects/` を確認（続きなら既存フォルダに積み上げる）
2. 新規なら次の番号でフォルダ作成: `projects/PRJ-NNN_案件名/`
3. `README.md` を作成する
4. **納期があれば Notion タスクDB に納期タスクを登録**する（下記ルール）

```text
projects/PRJ-001_案件名/
├── README.md       ← 依頼内容・ステータス・進捗ログ
├── output/         ← 成果物
├── resources/      ← 参考資料
└── notes/          ← 作業メモ
```

## README.md フォーマット

```markdown
# PRJ-NNN 案件名
- **依頼日**: YYYY-MM-DD
- **納期**: YYYY-MM-DD（なければ未定）
- **ステータス**: 未着手 / 進行中 / レビュー待ち / 完了
- **依頼内容**: （原文の要約）

## 進捗ログ
- YYYY-MM-DD: 何をしたか
```

## Notion 納期タスク連携

プロジェクト納期は朝ブリーフィングのデータ源（Notion タスクDB）経由で拾う。daily-briefing に `projects/*/README.md` の走査は含めない。

1. **立ち上げ時・納期設定/変更時**: Notion タスクDBへ納期タスクを登録または更新
   - 名前: `[PRJ-NNN] 案件名 — 納期`
   - ステータス: `未着手`（進行中なら `進行中`）
   - 優先度: 納期当日 → `P1-今日`、今週以内 → `P2-今週`、それ以外 → `P3-後で`
   - 期限: プロジェクトの納期
   - ソース: `プロジェクト`
   - メモ: `projects/PRJ-NNN_案件名/README.md` への参照
2. **完了・アーカイブ時**: 対応する Notion タスクのステータスを `完了` に更新

## 運用ルール

- 成果物の修正版は `output/` 内で `v1_`, `v2_` とバージョン番号を付けて残す（上書きしない）
- 「プロジェクト一覧を見せて」→ 全 `projects/*/README.md` を読み、番号・タイトル・ステータス・納期のテーブルで表示
- 「タスク一覧を見せて」→ Notion タスクDBの未完了タスクを優先度・期限順で表示
- 「〇〇は完了で」→ README のステータスを完了に更新、フォルダごと `archive/` へ移動、Notion タスクを `完了` に更新

## フォールバック

Notion MCP 障害時は納期タスクを `areas/tasks-backup.md` に追記し、復旧後にNotionへ移行する。
