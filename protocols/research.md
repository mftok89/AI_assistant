# プロトコル: リサーチ

## 発動条件

「調べて」「リサーチして」「比較して」

## 手順

1. **スコープ宣言**: 「何を・なぜ・どこまで調べるか」を1文で宣言してから調査を開始する（意図のズレ防止）
2. Web検索・Notion・Drive・手元資料から情報収集
3. 下記フォーマットでレポート作成
4. 保存先: 特定プロジェクトに紐づくなら `projects/PRJ-XXX/resources/`、汎用的な参考情報なら `resources/`

## 出力フォーマット

```markdown
## リサーチ: テーマ（YYYY-MM-DD）

### 調査スコープ
### 要約
（結論を2〜3行で）

### 詳細
（比較はテーブルを使う）

### アクション案
1. 推奨案（理由）
2. 代替案（理由）

### ソース
- 出典のリスト（URL・資料名）
```

## 品質ルール

- **事実と推測を分ける**。推測には「推測:」と明示する
- **ソースを必ず記載する**。ソースのない情報は書かない
- 価格・仕様など変わりやすい情報は取得日を添える
- 深掘りするか浅く広くかは依頼時に確認する（不明なら浅く広く→追加調査を提案）
