# プロトコル: 朝ブリーフィング

## 発動条件

「おはよう」「今日の予定」「ブリーフィング」

## 目的

カレンダー・メール・Notionタスクを統合し、今日1日の全体像を30秒で把握できるレポートを出す。

## 手順

1. **Google カレンダー**（MCP）: 今日の予定を取得
2. **Gmail**（MCP）: 未読・要対応メールを件数と概要で取得（本文全文は不要）
3. **Notion タスクDB**（MCP）: `config/tools.md` のDBを参照し、未完了タスク（ステータス ≠ 完了）を取得
   - 優先度 `P1-今日` または期限が今日のもの → 「今日やること」に
   - 期限超過 → 「期限切れ」として先頭に警告
   - ソース `プロジェクト` のタスクは project-mgmt 登録分（プロジェクト納期）。`projects/*/README.md` の走査は不要
4. 下記フォーマットで出力する（結論ファースト）
5. 要約を `areas/daily-briefings/YYYY-MM-DD.md` に保存する

## 出力フォーマット

```markdown
## 今日のポイント
（今日いちばん重要なこと・判断が必要なことを2〜3行）

## 期限切れタスク（あれば先頭に警告）
- タスク名（期限 M/D、N日超過）

## スケジュール
- HH:MM 予定名（準備が必要なら一言）

## 今日やること（Notionタスク: P1 + 期限今日）
- タスク名（ソース）

## メール要約
- 未読N件。要対応TOP3:
  - 「件名」from 差出人 — 要点と推奨アクション

## 判断待ち
- AI秘書が判断できなかった事項（なければ「なし」）
```

## フォールバック

- **Notion MCP 障害時**: `areas/tasks-backup.md` があれば参照。なければ「Notion 未接続。タスク確認は手動で」と明示
- **カレンダー/Gmail 障害時**: 「内容を貼り付けてもらえますか？」と依頼し、貼り付けテキストから同じフォーマットで作成

## 注意

- 30秒で読める分量に収める。詳細は聞かれたら出す
- 「判断待ち」は必ず設ける。理さんが考えるべきことだけを浮かび上がらせるのがこのレポートの価値
