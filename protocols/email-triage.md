# プロトコル: メールトリアージ

## 発動条件

「メール確認」「メールチェック」「メール整理して」

## 大原則

**メールの送信は絶対にしない。** 対応はGmailの下書き作成まで。送信は理さんが自分で行う。

## 優先度分類

| 段階 | 基準 | 対応 |
|------|------|------|
| P1 | 今日中に返信・対応が必要（期限あり・金銭関連・クレーム） | 返信下書き作成 + Notionタスク（P1-今日） |
| P2 | 今週中に対応 | 返信下書き or 返信方針提示 + Notionタスク（P2-今週） |
| P3 | 情報共有・後でよい | 要約のみ。タスク化は理さん確認後 |
| P4 | 不要・スパム | 一覧に載せるのみ |

## 手順

1. **Gmail**（MCP）: 未読・直近24時間のメールを取得
2. 上記4段階で分類し、P1/P2 を先に提示
3. P1/P2 について **Gmailに返信下書き**を作成（送信はしない）
   - 文体は `config/profile.md` と `memory/people/` の相手情報に合わせる
   - 判断が必要な内容（金額・可否・日程確定）は本文中に【要確認: 〇〇】と明示し、勝手に確定しない
4. **Notion タスクDB**（MCP）: P1/P2 の各メールについて、重複がなければタスクを追加
   - 名前: `[メール] 件名の要約`
   - ステータス: `未着手`
   - 優先度: P1 → `P1-今日`、P2 → `P2-今週`
   - 期限: P1 → 今日、P2 → 今週金曜（明確な期限があればそちら）
   - ソース: `メール`
   - メモ: 送信者・件名・メールURL（取得可能なら）
5. **重複チェック**: 既存Notionタスクと件名・送信者が重複する場合は新規作成せず、既存タスクを案内
6. 差出人が新規の関係者なら `memory/people/` に記録

## 出力フォーマット

```markdown
## メールトリアージ（YYYY-MM-DD）

### P1（今日中: N件）
1. 「件名」from 差出人（受信時刻）
   - 要点:
   - 推奨:
   - → 返信下書きをGmailに作成済み（送信前に確認してください）／Notionタスク登録済み

### P2（今週中: N件）
### P3（確認のみ: N件）
### P4（N件除外）
```

## フォールバック

- **Notion MCP 障害時**: タスクを `areas/tasks-backup.md` に追記し、「Notion 接続後に移行してください」と報告
- **Gmail MCP 障害時**: メール本文の貼り付けを依頼し、同様に分類・下書き文面を提示
